# 外部API連携ルール

## モデル・バージョン指定
- 外部APIのモデルIDやバージョンを指定する際は、必ず公式ドキュメントまたはWebSearchで最新の値を確認してから実装すること
- 古いモデルIDはAPI側で404になるリスクがある
- モデルIDは定数として切り出し、変更箇所を1箇所に集約する

## APIキー管理
- 環境変数は `server/lib/config/env.ts` の `getRequiredEnv()` / `getOptionalEnv()` 経由で取得する
- `process.env` を直接参照しない
- .env にハードコードされたキーをコードに含めない

## リトライ処理
- 外部APIへのHTTP呼び出しには必ずリトライ処理を実装する
- リトライ対象: ネットワークエラー、429(Rate Limit)、500/502/503/504(サーバーエラー)
- リトライ対象外: 400(Bad Request)、401(Unauthorized)、403(Forbidden)、404(Not Found) 等のクライアントエラー
- 最大リトライ回数: 3回（初回 + 2回リトライ）
- バックオフ: 指数バックオフ（1秒 → 2秒 → 4秒）
- リトライ用のユーティリティ関数を `server/utils/retry.ts` に集約し、各Strategy実装から呼び出す

## エラーハンドリング
- 外部API呼び出しは必ず try/catch で囲む
- APIレスポンスのHTTPステータスを確認し、非200はエラーとして処理する
- エラーメッセージにはプロバイダー名と操作内容を含める（例: "Groq Whisper STT変換に失敗しました"）
- 内部エラーの詳細はクライアントに露出しない
- リトライ上限を超えた場合は最後のエラーをそのまま throw する
